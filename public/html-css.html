HTML/CSS

HTML (HyperText Markup Language) — це мова розмітки, яка використовується для створення структури веб-сторінок. Веб-сторінка є набором елементів HTML, кожен з яких визначає, як повинні виглядати і функціонувати різні частини веб-контенту. Без HTML жоден веб-сайт не може існувати.

Уявімо HTML як каркас будівлі. Це структура, яка визначає, де будуть розташовані заголовки, текст, зображення, форми тощо. Кожен з цих елементів описується за допомогою тегів.

Основи HTML

Теги HTML

Основні елементи HTML визначаються тегами. Теги — це ключові частини HTML, які обмежують і формують вміст. Вони оточують текст або інший контент і вказують, як цей контент повинен виглядати або функціонувати. Кожен тег зазвичай має відкривальний та закривальний варіант (наприклад, <p> і </p>), хоча існують і одинарні теги (наприклад, <img>).

У HTML також існують атрибути, які додають додаткову інформацію чи налаштування для конкретного елемента (наприклад, src для зображень або href для посилань). Використання тегів за призначенням робить ваш код семантичним і зрозумілим, а також полегшує сприйняття сторінки пошуковими системами та технологіями доступності.

Ось приклад базової HTML-структури: 


<!DOCTYPE html>
<html lang="uk">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Моя перша веб-сторінка</title>
  </head>
  <body>
    <h1>Вітаю на моїй веб-сторінці</h1>
    <p>Це мій перший абзац тексту на сторінці.</p>
  </body>
</html>

Цей код містить основні теги:
- <!DOCTYPE html> оголошує, що це документ HTML5.
- Теги <html> і </html> охоплюють всю веб-сторінку.
- Теги <head> і </head> містять інформацію про сторінку (метадані, назва тощо).
- Теги <body> і </body> охоплюють видимий вміст сторінки, такий як заголовки, абзаци, зображення та інші елементи.

У цьому прикладі файл index.html описує структуру сторінки, де <head> містить метадані й заголовок, а <body> – усе, що бачить користувач. Дотримання правильної ієрархії (<h1>, <h2>, <p> тощо) допоможе надалі зручно стилізувати й підтримувати сайт.

Структурні теги

  • <h1> - <h6>: Заголовки різних рівнів
  • <p>: Тег для абзацу тексту
  • <a>: Тег для створення посилання
  • <img>: Тег для вставки зображення

Використовуючи ці теги, ви формуєте базову семантику сторінки. Наприклад, <h1> зазвичай використовується для основного заголовка, <h2> і <h3> – для підзаголовків, а <p> призначений для блоків звичайного тексту. Така впорядкованість сприяє кращій читабельності коду і простоті навігації.

Приклад додавання зображення та посилання 


<img src="image.jpg" alt="Опис зображення">
<a href="https://example.com">Перейти на інший сайт</a>

Тут <img> відображає зображення, а атрибут alt надає текстовий опис для користувачів із порушеннями зору або у випадках, коли зображення не завантажилося. Тег <a> створює гіперпосилання, дозволяючи користувачам переходити на зовнішні чи внутрішні сторінки.

HTML забезпечує лише базову структуру вашої веб-сторінки, але, щоб надати їй візуальну привабливість і гнучкий макет, ви використовуєте CSS, про який поговоримо далі.

Вступ до CSS

Що таке CSS?

CSS (Cascading Style Sheets) — це мова стилізації, що дає змогу задавати оформлення для елементів HTML. Якщо HTML — це «скелет» веб-сторінки, то CSS можна порівняти з «шкірою» та «одягом», який надає вашому сайту привабливого вигляду й зручності взаємодії.

Завдяки CSS ви можете керувати розташуванням блоків, кольорами, шрифтами, відступами, анімаціями і навіть робити адаптивний дизайн для мобільних пристроїв. Правильне використання CSS дозволяє відділити стиль від структури: HTML відповідає за розмітку, а CSS — за візуальне оформлення.

Основні способи підключення CSS

Існує три основні способи підключення CSS до HTML-документа: inline-стилі, внутрішні стилі та зовнішні стилі. У реальних проєктах найчастіше використовують зовнішні файли .css, але решта методів теж іноді стають у пригоді.

Inline-стилі

Застосовуються прямо в атрибуті style конкретного елемента. Хоча такий підхід зручний для швидких правок, зайве використання inline-стилів ускладнює подальшу підтримку й часто суперечить принципам розділення коду на структурний (HTML) та стилізувальний (CSS). 


<p style="color: blue;">Цей текст буде синім</p>

Внутрішні стилі

Це стилі, що прописані між тегами <style> всередині <head> вашої HTML-сторінки. Вони дозволяють описувати стилі для всієї сторінки, але якщо сторінок багато — доведеться дублювати код або підключати стилі в кожній окремо. 


<style>
  p {
    color: red;
  }
</style>

У невеликих проєктах внутрішні стилі можуть бути достатні, однак у більш масштабних — застосовувати одне окреме джерело стилів (зовнішній .css) набагато зручніше і зменшує дублювання.

Зовнішні стилі

Цей варіант найчастіше використовується у професійній розробці: ви створюєте файл styles.css і підключаєте його у <head> HTML-документа за допомогою: 


<link rel="stylesheet" href="styles.css">

Така практика полегшує командну роботу, адже кілька людей можуть одночасно оновлювати стилі й HTML окремо. Крім того, браузери кешують файл .css, що прискорює завантаження сайту. Це дуже важливо для продуктивності та зручності користувачів.

Синтаксис CSS

У CSS є «селектор», який визначає, до якого саме елемента (чи групи елементів) застосовувати стилі, і «декларації» у фігурних дужках, що складаються з «властивості» та «значення». Наприклад: 


h1 {
    color: green;
    font-size: 24px;
}

p {
    color: gray;
    line-height: 1.5;
}

Якщо ви хочете, щоб усі посилання <a> були синіми і не мали підкреслення, можна написати: a { color: blue; text-decoration: none; }. Завдяки гнучкій системі селекторів (класи, ідентифікатори, атрибути) ви можете дуже точно налаштовувати стиль кожного елемента.

Каскадність у CSS

«Каскадність» означає, що якщо на елемент діє кілька правил, то їх порядок і специфічність визначають, яке з них «переможе». Є три основні рівні:

  • Inline-стилі (найвищий пріоритет)
  • Внутрішні стилі
  • Зовнішні стилі

Також селектори можуть мати різний «ваговий коефіцієнт» (специфічність): наприклад, #header .menu li a детальніший, ніж просто a. Це означає, що за конфлікту останній (детальніший) селектор матиме вищий пріоритет.

Практика та поради

Стилізація тексту за допомогою CSS

Нижче наведено кілька ключових властивостей для оформлення тексту, які зустрічаються в більшості сайтів:

color: Визначає колір тексту 


p {
    color: blue;
}

Колір можна задавати назвою (blue), шістнадцятковим кодом (#0000ff) чи функціями rgb()/rgba(). Вибір кольорів сильно впливає на зручність читання та загальний дизайн.

font-size: Задає розмір шрифту 


p {
    font-size: 18px;
}

Розмір шрифту також можна встановлювати в em, rem, відсотках тощо. Використання відносних одиниць часто допомагає зробити сайт адаптивним до різних розмірів екрана й налаштувань браузера.

font-family: Шрифт тексту


p {
    font-family: Arial, sans-serif;
}

Щоб шрифт відображався коректно у всіх користувачів, зазвичай задають «стек» шрифтів: якщо немає першого, використовується наступний. Наприклад: font-family: 'Open Sans', Arial, sans-serif;.

text-align: Вирівнювання тексту 


h1 {
    text-align: center;
}

Значення можуть бути left, right, center та justify (вирівнювання за шириною). Заголовки нерідко вирівнюють по центру, а звичайний текст — за лівим краєм чи «в ширину».

Оформлення блоків і фонів

Крім управління текстом, CSS дозволяє «прикрашати» блоки й задавати кольори чи зображення фону, відступи та рамки.

background-color: Колір фону 


body {
    background-color: lightgray;
}

Можна також використовувати background-image, background-repeat, background-position для більш складних ефектів. Наприклад, задати градієнт або декоративне зображення.

padding: Внутрішні відступи


div {
    padding: 20px;
}

padding створює простір між вмістом і рамкою елемента. Якщо вам треба відстань між різними елементами, використовуйте margin. А для власне рамки є border.

border: Рамка елементів


div {
    border: 2px solid black;
}

border дозволяє задати товщину, стиль (solid, dotted) і колір. Використовуючи border-radius, можна зробити заокруглення кутів (наприклад, для кнопок чи карток).

Створення макета за допомогою CSS

Сучасні веб-сторінки часто використовують Flexbox чи CSS Grid для побудови макета. Це потужні інструменти, які дозволяють легко створювати складні, адаптивні дизайни.

Є також позиціонування через position: absolute;, position: relative; тощо, але воно частіше застосовується для дрібних деталей (наприклад, іконок, «спливаючих» вікон). У більшості випадків flex чи grid спрощують розробку.

Блокова модель

Кожен елемент на сторінці можна розглядати як прямокутну «коробку» (box). Усередині цієї коробки є вміст, навколо нього — padding, далі border, а вже зовні — margin. Розуміючи цю модель, ви краще контролюєте зовнішній вигляд елементів, їх відступи й суміжні області.

Flexbox: Сучасний спосіб гнучких макетів

Приклад вертикального вирівнювання:


.container {
    display: flex;
    justify-content: center;
    align-items: center;
}

За допомогою Flexbox ви можете вказувати напрям (row/column), упорядковувати елементи, розподіляти простір між ними та вирівнювати їх по центру. Це значно полегшує створення адаптивних макетів без громіздких «плаваючих» блоків. Для ще складніших двовимірних макетів використовується CSS Grid.

Підсумок

HTML та CSS є фундаментом для створення веб-сторінок. HTML формує структуру, а CSS відповідає за оформлення. Щоб стати кваліфікованим веб-розробником, потрібно навчитися комбінувати ці технології та використовувати їх для створення сучасних, зручних та естетично привабливих веб-сайтів. Дуже важливо пам'ятати про семантику HTML — це покращує доступність і SEO.

Надалі, ви можете додавати JavaScript, створювати адаптивні макети під мобільні пристрої, або навіть застосовувати фреймворки (Bootstrap, Tailwind, тощо). Але в основі залишається вміле використання базових інструментів — розмітки HTML і стилізації за допомогою CSS.
